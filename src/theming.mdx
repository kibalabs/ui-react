# Theming

UI-React uses a pure SCSS/CSS approach to theming, leveraging CSS custom properties (variables) for maximum flexibility and performance.

## Design Principles

### Ease of Use

UI-React is designed to be plug and play. Install the library, import the styles, and start using components immediately:

```tsx
// Import base styles
import '@kibalabs/ui-react/dist/index.css';

// Use components
<Button text="Click me" />
```

No configuration required for basic usage. Customization is opt-in and incremental.

### Consistency

All components share a common design language through CSS custom properties. When you change `--kiba-color-brand-primary`, every component using that color updates automatically.

Variants ensure that component variations are intentional and reusable:

```tsx
// These will look consistent across your entire app
<Button variant="primary" text="Save" />
<Button variant="secondary" text="Cancel" />
```

### Cascading Customization

CSS custom properties naturally cascade, enabling powerful theming patterns:

```scss
// Global theme
:root {
  --kiba-color-brand-primary: #4361ee;
}

// Dark mode section
.dark-theme {
  --kiba-color-brand-primary: #7c8aff;
  --kiba-color-background: #1a1a2e;
  --kiba-color-text: #ffffff;
}

// Component-specific override
.special-section {
  --kiba-color-brand-primary: #e63946;
}
```

## Setting Up Theming

### 1. Import Base Styles

```tsx
// Option A: Import compiled CSS
import '@kibalabs/ui-react/dist/index.css';

// Option B: Import SCSS (for customization)
@use '@kibalabs/ui-react/styles';
```

### 2. Define Your Theme

Create a theme file with your brand colors and preferences:

```scss
// theme.scss
:root {
  // Brand colors
  --kiba-color-brand-primary: #your-primary-color;
  --kiba-color-brand-secondary: #your-secondary-color;

  // Text colors
  --kiba-color-text: #1a1a2e;
  --kiba-color-text-light: #6b7280;
  --kiba-color-text-on-brand: #ffffff;

  // Background colors
  --kiba-color-background: #ffffff;
  --kiba-color-background-light: #f9fafb;

  // Typography
  --kiba-font-family: 'Your Font', system-ui, sans-serif;
  --kiba-font-size-base: 16px;

  // Spacing scale
  --kiba-spacing-unit: 4px;

  // Border radius
  --kiba-border-radius: 6px;
  --kiba-border-radius-lg: 12px;
}
```

### 3. Import in Your App

```tsx
// App.tsx
import '@kibalabs/ui-react/dist/index.css';
import './theme.scss';
```

## Dark Mode

Implement dark mode by overriding variables in a class or media query:

```scss
// Class-based dark mode
.dark-theme {
  --kiba-color-background: #1a1a2e;
  --kiba-color-background-light: #2d2d44;
  --kiba-color-text: #ffffff;
  --kiba-color-text-light: #a0a0b0;
  --kiba-color-border: #3d3d5c;
}

// System preference dark mode
@media (prefers-color-scheme: dark) {
  :root {
    --kiba-color-background: #1a1a2e;
    --kiba-color-text: #ffffff;
    // ... other dark mode overrides
  }
}
```

## Component Variants

Each component supports variants for different visual styles:

```tsx
// Button variants
<Button variant="primary" />
<Button variant="secondary" />
<Button variant="tertiary" />
<Button variant="small" />
<Button variant="large" />

// Combine variants
<Button variant="primary-large" />
```

### Creating Custom Variants

Add your own variants by extending component styles:

```scss
@layer kiba-theme {
  // Add a "ghost" button variant
  .KibaButton.ghost {
    background-color: transparent;
    border: 1px solid var(--kiba-color-brand-primary);
    color: var(--kiba-color-brand-primary);

    &:hover {
      background-color: var(--kiba-color-brand-primary);
      color: var(--kiba-color-text-on-brand);
    }
  }
}
```

## Best Practices

1. **Use CSS variables** - Always reference `--kiba-*` variables instead of hardcoded values
2. **Leverage cascading** - Use parent classes to scope theme changes
3. **Create variants** - Instead of one-off styles, create reusable variants
4. **Use layers** - Place customizations in `@layer kiba-theme` for proper specificity
5. **Keep it semantic** - Name variables by purpose, not appearance (e.g., `--kiba-color-error` not `--kiba-color-red`)
